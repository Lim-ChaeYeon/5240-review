# 🚀 팀원과 공유하기 - 배포 가이드

## 가장 쉬운 방법: GitHub Pages

### 📌 설정 순서 (3분이면 완성!)

#### 1단계: GitHub 저장소 생성

1. GitHub 접속 (https://github.com)
2. 우측 상단 **"+"** 클릭 → **"New repository"**
3. 설정:
   ```
   Repository name: review
   Description: 2025년 팀 결산
   Public ✅ (중요!)
   ```
4. **"Create repository"** 클릭

---

#### 2단계: 파일 업로드

**방법 A: 터미널 사용** (추천)

```bash
cd /home/sdh/5240/review

# Git 초기화
git init
git add .
git commit -m "팀 2025년 결산 페이지"

# GitHub 연결 (YOUR_USERNAME을 본인 것으로 변경!)
git remote add origin https://github.com/YOUR_USERNAME/review.git
git branch -M main
git push -u origin main
```

**방법 B: 웹에서 직접 업로드**

1. GitHub 저장소 페이지에서 **"uploading an existing file"** 클릭
2. 모든 파일 드래그 앤 드롭
3. **"Commit changes"** 클릭

---

#### 3단계: GitHub Pages 활성화 ⭐ 핵심!

1. GitHub 저장소에서 **"Settings"** 탭 클릭
2. 왼쪽 메뉴에서 **"Pages"** 클릭
3. **"Source"** 섹션에서:
   ```
   Branch: main
   Folder: / (root)
   ```
4. **"Save"** 클릭
5. 1-2분 대기 (자동 배포 중...)
6. 페이지 상단에 주소 표시됨:
   ```
   ✅ Your site is live at https://YOUR_USERNAME.github.io/review/
   ```

## 💡 장점

### ✅ 팀원 입장

- **클릭 한 번**으로 바로 확인
- **설치 불필요** (브라우저만 있으면 OK)
- **모바일에서도** 동일하게 접속
- **항상 최신 버전** (자동 업데이트)

### ✅ 관리자 입장

- **무료** (GitHub Pages는 완전 무료)
- **자동 배포** (git push만 하면 끝)
- **간단한 수정** (data.js 수정 → push → 자동 반영)
- **버전 관리** (Git으로 변경 이력 추적)

## 🔐 비공개로 공유하고 싶다면?

### GitHub Private Repository + Pages

1. 저장소를 Private로 생성
2. Settings → Pages 활성화
3. Settings → Collaborators
   → 팀원 이메일 추가

**주의**: GitHub Free는 Private 저장소의 Pages가 제한적  
→ **Netlify 비밀번호 보호** 사용 추천

### Netlify 비밀번호 보호

```
Netlify 배포 후:
Settings → Access control → Password protection
→ 비밀번호 설정
→ 팀원들에게 비밀번호 공유
```

## 🆘 문제 해결

### "404 Not Found" 나올 때

**원인**: GitHub Pages 배포 중 (1-2분 소요)  
**해결**: 2-3분 기다린 후 새로고침

### "Repository is private" 오류

**원인**: Private 저장소는 Pages 제한  
**해결**: 저장소를 Public으로 변경

```
Settings → Danger Zone → Change visibility → Public
```

### 수정했는데 반영 안 될 때

```bash
# 브라우저 캐시 삭제
Ctrl + Shift + R (강력 새로고침)

# 또는
Settings → Pages → "Re-run workflow"
```
